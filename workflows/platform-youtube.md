# Workflow — Platform: YouTube

## Role in system
Long-form authority channel. Build Receipts live here as full walkthroughs. Loom recordings are the raw asset — YouTube is one downstream channel from the Loom pipeline. Also produces YT Shorts (sub-60s) clipped from longer content.

## Cadence
- Long-form: 1/week (10-20 min). Starts Day 9-10 (Phase 2).
- YT Shorts: 1/week (repurposed from Loom or existing long-form). Starts Day 5 (Phase 1).
- Phase 1 (Days 1-7): Shorts only. Long-form begins Phase 2.

## Format rules

### Long-form (10-20 min)
```
Hook (0-30s): What you'll get. Specific outcome. No intro music, no "welcome back".
Problem (30s-2min): What the pain looks like before the automation. Name the hours/rupees.
Build walkthrough (2min-15min): Screen recording. Narrate what you're doing and why.
Result (15min-17min): Before vs after. Numbers only.
CTA (final 30s): Subscribe + one ask (DM for the workflow / comment your use case).
```

Script generated by `video-script-generator` skill. Timestamps are targets, not hard stops.

### YT Shorts (< 60s)
- Hook in first 3 seconds — spoken, not text overlay.
- One idea only. No sub-points.
- End with a hook to the long-form if available: "Full workflow in the description."
- Vertical format (9:16). Captions required (auto-generated + reviewed).

### Thumbnail rules
- Text: max 5 words, large font.
- Before/after structure whenever possible: "8 HRS → 2 MIN".
- Face in thumbnail performs better than screengrabs — use if comfortable.
- Do NOT use misleading thumbnails. Respect the viewer's time.

## Loom → YouTube pipeline
1. Record Loom (3-5 min for pillar content).
2. `loom-to-multipost` skill: pulls transcript + metadata.
3. `repurpose_loom_to_multipost.py`: produces LinkedIn post + X thread + carousel draft + YT short (clipped or re-recorded shorter).
4. Full Loom = embedded in blog draft + linked from LinkedIn post description (first comment).
5. If Loom exceeds 5 min free tier: upload to YouTube unlisted → use as long-form asset directly.

## Publishing
Via `tools/youtube_upload.py` (direct YT Data API, Phase 3+).
Phase 1-2: upload manually via YouTube Studio. Log URL to `data/posts.json` after publish.

## SEO basics
- Title: "[Outcome] with [Tool] — [Time saved]" e.g. "Stock analysis in 2 min with n8n"
- Description: first 2 lines are what appears in search — put the outcome here, not "in this video".
- Tags: tool names + use case + "India" where relevant.
- Chapters: add timestamps manually after upload (helps watch time and SEO).
