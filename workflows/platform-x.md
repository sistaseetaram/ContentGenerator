# Workflow — Platform: X (Twitter)

## Role in system
Daily takes and thread repurposing. Feeds into LinkedIn (X is the testing ground for hooks — high engagement on X → expand into LinkedIn post). Plain-English AI Takes primary home. Build Receipts as X threads (repurposed from LI, not copy-paste).

## Cadence
- Target: 2-3 posts/day (mix of original + replies + quote-tweets)
- Minimum: 1 original post/day (no days off once X publishing is live, Day 11+)
- Threads: 1/week (repurposed Build Receipt or expanded Plain-English Take)
- Phase 1 (Days 1-10): X not live yet. LinkedIn only.

## Format rules

### Single tweet (280 chars)
- Hook + insight in one shot. Complete thought.
- No "thread incoming 🧵" for single tweets — that's a thread promise, deliver or don't use it.
- End with a one-word hook or question only if natural. Not forced.

### Thread format
```
Tweet 1: Hook — the payoff upfront. Strongest line.
Tweet 2-N: Each tweet = one idea. Self-contained if pulled out of context.
Final tweet: Summary or CTA. "If this was useful, repost."
```

### Thread length
- Build Receipts threads: 5-8 tweets
- Plain-English Takes threads: 3-5 tweets
- Never pad to hit a number. If the thread ends at tweet 4, it ends at tweet 4.

### Repurposing rule
LinkedIn post → X thread: rewrite the hook for X (shorter, snappier). Do NOT copy-paste. X audience is different — less formal, faster-paced, skims harder.

## Voice rules
Same core voice as LinkedIn: outcome first, real numbers, no jargon, short sentences.
X-specific adjustments:
- Hook can be blunter than LinkedIn. X rewards directness.
- Emojis: 0-1 per tweet, only if they genuinely add. Not for decoration.
- Hashtags: 0-1 max. Twitter hashtags have minimal reach effect — skip unless trending relevant topic.

## Publishing (Phase 1-2)
Via Blotato. Connect X account. Schedule or post immediately.

## Publishing (Phase 3+)
Via `tools/x_post.py` (direct X API). Blotato parallel for 2-week validation.
