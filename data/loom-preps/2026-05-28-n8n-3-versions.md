## Content Guidance

What to say, what not to say, and why — for this specific recording.

### ✅ Mention these

- **All 3 versions on screen** — don't describe them, show them. Toggle between tabs. The visual diff is the point.
- **Exact cost numbers: $1 → $0.06, 94% reduction** — say these out loud. Slow down on them. This is why people watch.
- **What specifically changed each version** — not vague ("it got better"), specific ("I pulled static company data outside the prompt, so v2's prompt was 70% smaller").
- **The real failures** — data mapping errors, variable names pointing nowhere, the node that ran forever. This is the brand. Build Receipts = show the honest work.
- **That v4 is coming** — explicitly say it. Creates continuity, gives reason to follow.
- **How long this took** — "weeks of following tutorials" before v1. The time cost is real and relatable.
- **Why each version existed** — don't just show the diff. Say the thought process: "I hit X problem, which led me to discover subworkflows."

### ❌ Don't mention

- **Specific stock names / tickers shown in the demo** — the automation is the point, not the stocks. Blur or replace with dummy data if needed.
- **Which data API you use for prices** — either not relevant to the lesson OR you don't want to advertise a free tier someone else will hit-rate-limit on.
- **Total cost to run n8n** (infrastructure, VPS, etc.) — different conversation, adds complexity, muddies the $0.06 clarity.
- **Comparisons to other tools** — "this beats Make/Zapier/etc." — not yet. You don't have the data. Contrarian pillar (Week 3+) is the right home for comparisons.
- **Apologies for v1** — "sorry it looks bad" or "I was still learning" = weak framing. Own v1 fully. It's the starting point, not a shame spiral.

## Talking points

### Beat 1: v1 — one massive prompt
- Show the v1 canvas zoomed out — every input feeds one prompt node
- Point at the one giant prompt: "static company facts + dynamic prices + analysis instructions, all in one envelope"
- Surface the cost number: "$1 per report" — say it slow
- Why this happens: it's how every tutorial demo looks. Works on demo. Breaks on volume.
- Don't apologize for it. This is the receipt — show the bad version.

### Beat 2: v2 — subworkflows + data split
- Open v2 canvas side-by-side with v1 if possible (split screen, or quick toggle)
- "Discovered subworkflows" — point at the new subworkflow nodes
- Static data (company name, sector, fundamentals) now lives outside the prompt
- Dynamic data (live prices) fetched per-call
- Prompt is now ~30% the size
- Cost dropped — quote the v2 cost if logged, else "significant drop, but not enough"

### Beat 3: v3 — verification before fetching
- Open v3 canvas
- Point at the new verification step before any data fetch
- "Model only sees what it needs"
- The number reveal: "$0.06 per report" — say it slow, let it land
- Cost reduction math: "$1 to $0.06 is 94%"
- This isn't a feature. This is just the architecture finally making sense.

### Beat 4: why this took 3 versions
- Be specific: "data mapping errors", "variable names pointing nowhere", "a node running forever on malformed input"
- "Nobody ships a good automation on the first try"
- "Most tutorials show the finished thing. They skip the three versions before it."
- "The frustration is the curriculum."
- Don't soften this. It's the real argument.
