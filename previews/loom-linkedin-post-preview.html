Same workflow. Three versions. 94% cheaper.

I recorded the walkthrough for my n8n stock analysis workflow.

Version 1 was one giant prompt.
Static company data, live prices, analysis rules, formatting rules. Everything went into one envelope.

It worked as a demo.
It cost about $1 per report.

Version 2 split the work into subworkflows: financials, stock data, index data, and formatting.

Better structure. Still not enough.

Version 3 added one small question before anything ran:

Do I actually need to generate this report again?

That changed the cost from $1 to $0.06 per report.
Same baseline model. Same output. 94% lower cost.

The useful part was not the final workflow.

It was the two bad versions before it.

Variable names pointing nowhere.
Data mapping errors.
Nodes running too long on malformed input.
Three nights lost to one character in one field name.

Most tutorials show the finished automation.
They skip the versions that taught you why it works.

That is the part I am trying to document.

V4 is already being rebuilt on a different architecture.

What version are you on?
